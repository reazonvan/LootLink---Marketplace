# LootLink Marketplace

## What This Is

LootLink — это P2P маркетплейс для торговли игровыми предметами, скинами, аккаунтами и внутриигровой валютой. Платформа соединяет продавцов и покупателей игровых товаров, обеспечивая безопасные сделки между игроками.

## Core Value

Уникальный, запоминающийся дизайн, который выделяет LootLink среди конкурентов (FunPay, PlayerOK, GGSell) и создает премиальное впечатление для пользователей игрового маркетплейса.

## Requirements

### Validated

- ✓ Django 5.2 backend с системой управления пользователями — existing
- ✓ PostgreSQL база данных — existing
- ✓ Базовый функционал товаров и листингов — existing
- ✓ Существующие frontend компоненты — existing

### Active

- [ ] Полный редизайн всего сайта с уникальной визуальной идентичностью
- [ ] Современный минималистичный дизайн интерфейса
- [ ] Уникальные дизайн-решения, которые запоминаются
- [ ] Сбалансированная визуальная плотность (избежать пустоты и перегруза)
- [ ] Премиальный внешний вид, отличающий от конкурентов
- [ ] Главная страница (landing) с сильным первым впечатлением
- [ ] Карточки товаров с уникальным дизайном
- [ ] Навигация и меню с современным UX
- [ ] Страницы каталога товаров
- [ ] Детальные страницы товаров

### Out of Scope

- Изменение backend логики — фокус только на frontend/дизайн
- Добавление новых функций маркетплейса — только визуальное улучшение существующего
- Миграция на другой tech stack — работаем с Django

## Context

**Тип проекта:** Brownfield — существующий Django маркетплейс требует полного визуального обновления

**Конкуренты:** FunPay, PlayerOK, GGSell — все имеют похожий функционал, но LootLink должен выделяться дизайном

**Текущие проблемы дизайна:**
- Выглядит устаревшим и неинтересным
- Не уникальный, похож на конкурентов
- Пустые страницы в некоторых местах
- Перегруженные страницы в других местах
- Нет запоминающихся дизайн-решений
- Не создает впечатление премиальности

**Целевая аудитория:** Геймеры, покупающие и продающие игровые предметы — ожидают современный, быстрый, визуально привлекательный интерфейс

**Желаемая эстетика:** Современный минимализм с уникальными акцентами

## Constraints

- **Tech stack**: Django 5.2, PostgreSQL — сохраняем существующий backend
- **Scope**: Только frontend/дизайн — backend логика не меняется
- **Совместимость**: Новый дизайн должен работать с существующими Django views и моделями
